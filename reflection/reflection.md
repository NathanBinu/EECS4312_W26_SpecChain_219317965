# Application: Mind Doc

- Differences between pipelines: The manual pipeline produced focused personas and requirements grounded directly in specific review groups, while the automated pipeline generated broader and more generalized personas. The hybrid pipeline combined both approaches by using automated generation with manual refinement to improve clarity and alignment with the data.

- Clearest personas: The manual pipeline produced the clearest personas because they were directly derived from carefully selected review groups, making them more specific and realistic. Automated personas were often broader and less precise.

- Most useful requirements: The hybrid pipeline produced the most useful requirements, as it balanced automation efficiency with manual refinement, resulting in clearer and more actionable requirements compared to the automated pipeline alone.

- Strongest traceability: The manual pipeline had the strongest traceability since each requirement could be clearly linked back to specific review groups. The automated pipeline sometimes weakened traceability due to generalized outputs, while the hybrid approach improved it but did not fully match the manual pipeline.

- Problems in automated outputs: The automated pipeline often generated requirements that were grammatically correct but too vague to support reliable test generation. Some personas also included assumptions that were not clearly supported by the original reviews, reducing accuracy and trustworthiness.
